There is one http kdb-srv object that uses Ajax to send requests and expects the result in json format. It also automatically fixes kdb 3.2 issue with "json?query"

([] a:1 2 3; b:`h`j`k; c:("a&a";"v%v";"l<b>BOLD</b>l"))

Both tables are linked to the same query but the second table does not escape html chars.

Http srv with json return type may have an optional k-deserialize option. In this case results like ("deserialize";`int$-8!expr) will be automatically deserialized via c.js

("deserialize";`int$-8!([] a:1 2 3; b:`h`j`k; c:("a&a";"v%v";"l<b>BOLD</b>l")))
